# 📖 API Documentation Template

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Version](https://img.shields.io/badge/version-1.0.0-blue.svg)](https://github.com/AhmarHussain/api-documentation-template)
[![OpenAPI](https://img.shields.io/badge/OpenAPI-6BA539?style=flat&logo=openapi-initiative&logoColor=white)](https://www.openapis.org/)

A premium, high-performance API documentation template built with **Redoc** and **Swagger UI**. Features a sleek dark mode, instant search, and a mobile-responsive layout to make your API easy to explore and integrate.

---

## 🚀 Why Use This Template?

Great APIs deserve great documentation. Most default Swagger UI setups look dated and lack the "premium" feel that developers expect. **API Documentation Template** provides a modern, fast, and highly customizable interface for your OpenAPI 3.0+ specs.

### Key Features:
- **Dual Display:** Includes both Swagger UI (interactive) and Redoc (three-column) versions.
- **Sleek Dark Mode:** Easy on the eyes for late-night integration sessions.
- **Instant Search:** Powerful fuzzy search across all endpoints and schemas.
- **Response Examples:** Beautifully formatted JSON/XML response bodies with copy-to-clipboard.
- **Code Snippets:** Automatic generation of `curl`, `javascript`, `python`, and `php` examples.
- **Fully Responsive:** Works perfectly on desktops, tablets, and mobile devices.

---

## 🛠️ Getting Started

1. Clone the repository.
2. Replace the `openapi.yaml` file with your own API specification.
3. Open `index.html` in your browser to preview.
4. Deploy the `dist/` folder to GitHub Pages, Netlify, or Vercel.

---

## 📖 Customization

You can easily theme the documentation by modifying the CSS variables in `assets/css/theme.css`.

```css
:root {
  --primary-color: #3498db;
  --bg-dark: #1a1a1a;
  --text-main: #f5f5f5;
}
```
